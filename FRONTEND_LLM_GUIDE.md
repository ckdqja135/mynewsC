# 프론트엔드 LLM 분석 기능 사용 가이드

Cerebras LLM을 활용한 뉴스 분석 기능이 프론트엔드에 통합되었습니다.

## 🚀 빠른 시작

### 1. Backend 서버 시작

```bash
cd backend

# 의존성 설치 (처음 한 번만)
pip install -r requirements.txt

# 서버 실행
uvicorn app.main:app --reload
```

서버가 `http://localhost:8000`에서 실행됩니다.

### 2. Frontend 서버 시작

```bash
cd frontend

# 의존성 설치 (처음 한 번만)
npm install

# 개발 서버 실행
npm run dev
```

프론트엔드가 `http://localhost:3000`에서 실행됩니다.

## 📱 사용 방법

### 기본 검색 페이지 (/)

1. 메인 페이지에서 뉴스 검색
2. 우측 상단의 **"🤖 AI 뉴스 분석"** 버튼 클릭

### 분석 페이지 (/analyze)

#### 1. 분석 설정

- **분석할 주제**: 분석하고 싶은 주제 입력 (예: 인공지능, 경제, 기후변화)
- **분석 유형**: 4가지 옵션 선택
  - **종합 분석** (추천): 감성 + 트렌드 + 핵심 포인트 모두 포함
  - **감성 분석**: 긍정/부정 측면과 감성 점수
  - **트렌드 분석**: 주요 토픽, 신흥 트렌드, 핵심 인물
  - **핵심 포인트**: 가장 중요한 정보만 간결하게
- **분석할 기사 수**: 10~50개 (20개 추천)

#### 2. 분석 결과 확인

**공통 정보:**
- 📋 요약: 전체 내용의 2-3문장 요약
- 🎯 핵심 포인트: 주요 인사이트 리스트

**종합 분석 / 감성 분석:**
- 💭 감성 분석
  - 전반적 감성 (positive/negative/neutral)
  - 감성 점수 (-1.0 ~ 1.0)
  - ✅ 긍정적 측면
  - ⚠️ 부정적 측면

**종합 분석 / 트렌드 분석:**
- 📈 트렌드 분석
  - 🏷️ 주요 토픽
  - 🚀 신흥 트렌드
  - 👥 핵심 인물/기관

## 📊 분석 시간

- **일반적인 경우**: 5-15초
- **많은 기사 (50개)**: 최대 2분
- 분석 중 로딩 스피너와 진행 메시지 표시

## 💡 사용 팁

### 최적의 분석 결과를 위한 팁

1. **명확한 주제 입력**: "AI"보다는 "인공지능 기술 발전"처럼 구체적으로
2. **적절한 기사 수**: 20개가 품질과 속도의 균형점
3. **분석 유형 선택**:
   - 처음 사용: 종합 분석
   - 여론 파악: 감성 분석
   - 트렌드 조사: 트렌드 분석
   - 빠른 정보: 핵심 포인트

### 예시 쿼리

- "인공지능 윤리"
- "반도체 산업 동향"
- "기후변화 대응"
- "K-POP 해외 진출"
- "전기차 시장"

## 🐛 문제 해결

### "분석에 실패했습니다"
- Backend 서버가 실행 중인지 확인
- Cerebras API 키가 `.env`에 올바르게 설정되었는지 확인
- 네트워크 연결 확인

### 분석이 너무 오래 걸림
- 기사 수를 줄여보세요 (10-20개)
- Backend 로그 확인: `uvicorn app.main:app --reload`

### CORS 에러
- Backend의 CORS 설정 확인
- `backend/app/main.py`에서 `allow_origins`를 확인
